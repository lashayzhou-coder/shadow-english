# 阶段 1 - P0 测试清单

**目标：验证音频播放器核心功能和 PWA 基础配置**

## 1. 项目初始化与构建
- [ ] 项目目录结构是否正确
- [ ] `npm install` 是否成功执行
- [ ] `npm run build` 是否成功执行
- [ ] `dist/` 目录是否包含正确的文件（index.html, assets, sw.js）

## 2. 音频播放器核心功能

### 2.1 音频源输入
- [ ] URL 输入框是否可正常输入
- [ ] 本地文件上传功能是否正常
- [ ] 音频源类型切换（URL/文件）是否正常

### 2.2 播放控制
- [ ] 播放/暂停功能是否正常
- [ ] 停止功能是否正常
- [ ] 快退 5 秒功能是否正常
- [ ] 快进 5 秒功能是否正常
- [ ] 进度条拖拽是否正常工作

### 2.3 音量控制
- [ ] 音量滑块是否可调节
- [ ] 音量显示是否正确更新

### 2.4 变速播放
- [ ] 0.5x / 0.75x / 1.0x / 1.25x / 1.5x 速度是否可切换
- [ ] 播放速度是否实时更新

### 2.5 AB 复读功能
- [ ] A 键（设置 A 点）是否正常
- [ ] B 键（设置 B 点）是否正常
- [ ] C 键（清除 AB 点）是否正常
- [ ] AB 区间是否在进度条上正确高亮
- [ ] AB 区间循环播放是否正常

### 2.6 键盘快捷键
- [ ] 空格键是否控制播放/暂停
- [ ] 左/右方向键是否控制快退/快进 5 秒
- [ ] A/B/C 键是否控制 AB 复读

## 3. 界面测试

### 3.1 移动端适配
- [ ] 页面在移动端是否正常显示
- [ ] 按钮尺寸是否符合触摸要求（≥44×44px）
- [ ] 布局是否紧凑合理

### 3.2 平板端适配
- [ ] 页面在平板（≥768px）上是否正常显示
- [ ] 布局是否符合两栏设计

### 3.3 基础样式
- [ ] 颜色是否符合设计规范（主色靛蓝，辅助橙色）
- [ ] 字体大小是否合适
- [ ] 交互反馈是否明显

## 4. PWA 功能

### 4.1 基础配置
- [ ] manifest.json 是否正确配置
- [ ] 图标引用是否正确
- [ ] Service Worker 是否正确生成

### 4.2 离线支持
- [ ] 静态资源是否被缓存
- [ ] 离线状态下是否显示友好提示
- [ ] 音频是否不被缓存

## 5. 通用测试

- [ ] 所有链接是否可点击
- [ ] 图片资源是否正确加载
- [ ] 所有可交互元素是否有反馈
- [ ] 页面是否有语法错误

## 测试方法

### 1. 本地测试
```bash
# 启动开发服务器
npm run dev

# 访问页面进行测试
# 地址：http://localhost:5173/
```

### 2. 生产版本测试
```bash
# 构建生产版本
npm run build

# 预览生产版本
npm run preview

# 访问页面进行测试
# 地址：http://localhost:4173/
```

### 3. 移动端测试
- 在移动设备浏览器中访问开发服务器
- 或使用浏览器开发者工具的移动端模拟功能

### 4. PWA 测试
- 在 Chrome/Edge 浏览器中访问页面
- 检查应用是否可安装（地址栏出现 + 图标）
- 安装后检查功能是否正常

## 预期结果

所有测试项都应通过，页面功能正常，无明显缺陷。音频播放器应能稳定播放音频，支持所有控制功能，PWA 配置应正确工作。
